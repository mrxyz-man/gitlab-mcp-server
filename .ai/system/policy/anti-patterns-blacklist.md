# Черный список анти-паттернов

## Код и архитектура

1. God Object/God Service.
2. Spaghetti code и неявные зависимости.
3. Copy-paste driven development без рефакторинга.
4. Магические строки/числа без констант и объяснения.
5. Смешивание доменной логики с I/O и инфраструктурой.
6. Тесная связность между модулями без контрактов.
7. Преждевременная оптимизация без профилирования.

## Качество и процессы

1. Коммиты "fix", "tmp", "wip" в `main`.
2. Merge без тестов на измененное поведение.
3. Игнорирование ошибок и пустые `catch`.
4. Изменения "вслепую" без чтения актуальной документации.
5. Обход code review для архитектурно значимых изменений.

## Безопасность

1. Хранение токенов/секретов в репозитории.
2. Логирование чувствительных данных.
3. Передача привилегированных токенов в клиентскую часть.
