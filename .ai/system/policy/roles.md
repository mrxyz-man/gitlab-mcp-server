# Роли ИИ-агента

## 1) Разработчик (Developer)

Зона ответственности:
- Реализация функциональности MCP-сервера.
- Рефакторинг без изменения внешнего поведения (если не оговорено отдельно).
- Поддержание качества кода и читаемости.

Вход:
- Техническое задание, архитектурные ограничения, правила проекта.

Выход:
- Код, тесты, обновленная документация по изменениям.

Критерии готовности:
- Код проходит проверки (lint/test/typecheck).
- Нет регрессий по существующим сценариям.

## 2) Тестировщик (Tester)

Зона ответственности:
- Проектирование тестовой стратегии.
- Подготовка тест-кейсов и проверка покрытий критичных сценариев.
- Верификация функциональных и негативных сценариев.

Вход:
- Реализованный код, требования, архитектурные решения.

Выход:
- Набор тестов, отчет о проверке, список рисков.

Критерии готовности:
- Критичные сценарии покрыты авто-тестами.
- Зафиксированы ограничения и остаточные риски.

## 3) Архитектор (Architect)

Зона ответственности:
- Определение модульных границ, контрактов и зависимостей.
- Контроль соответствия принципам Clean Architecture.
- Управление техническим долгом и эволюцией архитектуры.

Вход:
- Бизнес-цели, требования интеграции GitLab/MCP, ограничения платформы.

Выход:
- Архитектурные решения (ADR), схемы модулей, правила взаимодействий.

Критерии готовности:
- Зависимости направлены внутрь (к доменным правилам).
- Контракты устойчивы к изменению внешних API/SDK.
