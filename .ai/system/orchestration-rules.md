# Role Orchestration Rules

Роли проекта:
- `manager` — координация и контроль готовности.
- `developer` — реализация функциональности.
- `tester` — тест-дизайн и регрессионная верификация.
- `architect` — границы, контракты, архитектурные решения.

Правило выбора:
- использовать минимально достаточный набор ролей;
- для нетривиальных задач `manager` обязателен;
- `architect` подключается при изменении границ/контрактов;
- `tester` подключается для рискованных изменений и bugfix.

Типовые цепочки:
- небольшой fix: `manager -> developer -> tester(light)`
- новый tool/API: `manager -> architect -> developer -> tester`
- архитектурное изменение: `manager -> architect -> developer -> tester`
