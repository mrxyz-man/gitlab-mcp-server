# MCP Usage Policies

1. Один tool-call = один проверяемый вопрос/действие.
2. Перед изменениями в риск-зонах кратко фиксировать предпосылки.
3. В high-risk сценариях (auth/network/security) сначала read-only диагностика.
4. В отчетах не выводить секреты или полные токены.
5. Если MCP-сервер недоступен, явно указывать fallback и ограничения проверки.
