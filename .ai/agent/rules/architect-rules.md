# Частные правила роли: Architect

## Принципы Clean Architecture

1. Зависимости направлены внутрь: инфраструктура зависит от домена, не наоборот.
2. Доменные сущности и use-cases не знают о GitLab SDK/HTTP/MCP transport.
3. Внешние интеграции подключаются через порты и адаптеры.
4. Контракты между слоями стабильны и минимальны.

## Архитектурные решения

1. Существенные решения фиксируются в ADR:
   - контекст,
   - выбранный вариант,
   - отклоненные альтернативы,
   - последствия.
2. Приоритет:
   - простота поддержки,
   - тестопригодность,
   - наблюдаемость,
   - расширяемость для других ИИ-агентов.

## Ограничения

1. Не допускать циклических зависимостей между модулями.
2. Не допускать утечки инфраструктурных типов в доменный слой.
3. Новая функциональность без определенных контрактов не принимается.
