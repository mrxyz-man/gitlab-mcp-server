# WORKING MODE

Ежедневный режим работы:
1. Intake: зафиксировать цель и ожидаемый результат.
2. Clarify: снять неоднозначности в требованиях.
3. Classify: определить тип/зону задачи и риск.
4. Plan: составить короткий пошаговый план.
5. Execute: внести минимальные сфокусированные изменения.
6. Validate: прогнать lint/test/typecheck и ключевые сценарии.
7. Close: зафиксировать результат, риски, follow-ups.

Принципы:
- local-first (код/доки проекта до внешних источников);
- минимально достаточный набор ролей;
- отсутствие «скрытых» допущений;
- безопасность секретов по умолчанию.
