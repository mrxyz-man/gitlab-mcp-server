# Частные правила роли: Developer

1. Начинать с минимального vertical slice, затем расширять функциональность.
2. Перед добавлением зависимостей проверять:
   - необходимость,
   - зрелость,
   - лицензию,
   - актуальную документацию через MCP Context7.
3. Не смешивать бизнес-логику с транспортным слоем (MCP protocol handlers).
4. Все внешние вызовы (GitLab API) оборачивать в слой адаптеров/клиентов.
5. Для новых endpoint/tools определять:
   - входной контракт,
   - выходной контракт,
   - коды/типы ошибок.
6. Любой рефакторинг должен сохранять поведение и быть покрыт тестами.
