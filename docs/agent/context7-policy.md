# Регламент использования MCP Context7

Цель: гарантировать, что технические решения опираются на актуальную документацию.

## Когда обязательно обращаться к Context7

1. Перед интеграцией новой библиотеки/SDK.
2. Перед изменением публичных API/контрактов.
3. При обновлении версий зависимостей.
4. При неочевидном поведении инструмента/фреймворка.

## Минимальный процесс

1. Найти библиотеку через `resolve-library-id`.
2. Запросить нужный раздел через `query-docs`.
3. Зафиксировать в задаче/PR:
   - какой источник использован,
   - какая версия/секция,
   - какое решение принято на основе источника.

## Запрещено

1. Внедрять нетривиальные решения без сверки с Context7.
2. Ссылаться на устаревшие или непроверенные гайды вместо официальной документации.
